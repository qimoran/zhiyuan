# 重庆高校志愿填报系统开发步骤文档

## 1. 文档说明

### 1.1 文档基本信息

| 项目 | 内容 |
| --- | --- |
| 文档名称 | 重庆高校志愿填报系统开发步骤文档 |
| 项目名称 | 重庆高校志愿填报系统 |
| 文档版本 | V1.0 |
| 编写日期 | 2026-06-15 |
| 依据文档 | `docx/详细设计文档.md` |
| 适用对象 | 项目组 4 名开发成员、测试成员、答辩准备人员 |
| 文档目标 | 将详细设计拆解为可分配、可测试、约 4 人天一个单位的开发步骤 |

### 1.2 拆分原则

1. 每个步骤工作量控制在 3.5 到 4.5 人天，默认由 1 名程序员独立负责。
2. 每个步骤必须有明确输入、输出、测试方式和完成标准。
3. 步骤之间按模块依赖关系排序，尽量减少返工。
4. 能并行的步骤明确标注并行条件。
5. 优先保证主链路可演示：建库建表、数据导入、位次查询、志愿推荐、页面展示、AI 报告。
6. Hive、Spark、真实爬虫和真实大模型接口作为增强能力，但需要保留可运行脚本和演示说明。

### 1.3 工作量说明

本文档中的“人天”表示 1 名开发人员 1 天的正常开发工作量。  
本项目建议拆为 15 个开发步骤，每个约 4 人天，总工作量约 60 人天，适合 4 人小组在 3 周左右完成。

## 5. 详细开发步骤

### S01 工程骨架与配置初始化

| 项目 | 内容 |
| --- | --- |
| 工作量 | 4 人天 |
| 前置依赖 | 无 |
| 适合负责人 | 后端开发或全栈开发 |
| 是否可并行 | 项目起点，不依赖其他步骤 |
| 对应详细设计 | 第 3 章、 第 4 章、 第 15 章 |

#### 目标

建立项目基础目录、Python 依赖、配置文件模板和最小运行环境，为后续所有模块提供统一结构。

#### 任务清单

1. 按详细设计创建 `src/`、`config/`、`sql/`、`scripts/`、`data/`、`logs/`、`tests/` 目录。
2. 创建 `requirements.txt`，包含 Flask、PyMySQL 或 SQLAlchemy、Pandas、NumPy、PyYAML、python-dotenv、pytest 等基础依赖。
3. 创建 `.env.example`，包含 MySQL、Redis、Hive、HDFS、大模型接口等配置项。
4. 创建 `config/config.yaml`、`config/recommend_rules.yaml`、`config/logging.yaml`。
5. 创建 `src/common/config_loader.py`，支持读取 `.env` 和 YAML 配置。
6. 创建 `scripts/` 下基础 PowerShell 脚本占位文件。
7. 编写最小 README 草稿，说明项目目录和后续运行方式。

#### 交付物

1. 项目目录结构。
2. `requirements.txt`。
3. `.env.example`。
4. `config/config.yaml`。
5. `config/recommend_rules.yaml`。
6. `src/common/config_loader.py`。
7. README 初稿。

#### 测试方法

```powershell
python -m pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
python -c "from src.common.config_loader import load_config; print(load_config())"
```

#### 完成标准

1. 目录结构与详细设计基本一致。
2. 依赖能正常安装。
3. 配置能被 Python 代码读取。
4. 不包含真实密码、Token、API Key。

## 8. 风险控制建议

| 风险 | 影响 | 控制方式 |
| --- | --- | --- |
| 真实数据采集失败 | 推荐和图表无真实数据 | S03 必须准备样例数据，先保证主链路 |
| Hive/Spark 环境不稳定 | 大数据分析无法演示 | S11 支持本地 CSV 输出，保留脚本和说明 |
| 推荐接口延期 | 前端页面无法联调 | 先用固定 JSON Mock，接口完成后替换 |
| 大模型接口不可用 | 报告无法生成 | S13 必须实现模板降级 |
| 四人进度不一致 | 最终联调压力大 | 每个步骤必须有测试命令，S15 前持续集成 |
| 文档与代码不一致 | 答辩和维护困难 | S15 统一更新 README 和测试记录 |
