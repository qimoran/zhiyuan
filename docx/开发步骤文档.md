### 1.1 文档基本信息

| 项目 | 内容 |

### 1.2 拆分原则

1. 每个步骤工作量控制在 3.5 到 4.5 人天，默认由 1 名程序员独立负责。
2. 每个步骤必须有明确输入、输出、测试方法和完成标准。
3. 步骤按项目依赖顺序排列，先完成基础设施、数据库、爬虫和数据闭环，再开发接口、推荐、前端、大数据和报告。
4. 能并行的步骤明确标注并行条件，减少 4 人小组等待时间。
5. 每个步骤都尽量可独立测试，不要求等到最后集成才发现问题。
6. 第一版优先保证可演示主链路：掌上考研院校库爬虫、候选库入库、官方资料登记、样例数据入库、推荐接口、页面展示、AI 报告和大数据分析链路。
7. 所有命令示例默认使用 Windows PowerShell。

### 1.3 工作量说明

本文档中的“人天”表示 1 名开发人员 1 天的正常开发工作量。  
本项目建议拆分为 15 个开发步骤，每个步骤约 4 人天，总工作量约 60 人天，适合 4 人小组在 3 周左右完成。

### S03 掌上考研院校库爬虫

| 项目 | 内容 |
| --- | --- |
| 工作量 | 4 人天 |
| 前置依赖 | S01 |
| 适合负责人 | 爬虫开发或数据采集成员 |
| 是否可并行 | 可与 S02 并行；入库功能依赖 S02 |
| 对应详细设计 | 第 6.1 节、第 12.2 节 |

#### 目标

实现第一版明确爬虫目标：从 `https://www.kaoyan.cn/school-list/50-0-0` 对应接口抓取重庆研招单位候选库，保存原始 JSON 并输出标准化 CSV。

#### 任务清单

1. 创建 `src/crawlers/kaoyan_school_list_crawler.py`。
2. 使用 `requests.post` 请求 `https://api.kaoyan.cn/pc/school/schoolList`。
3. 请求体固定包含 `province_id=50`，分页参数为 `page` 和 `limit`。
4. 自动分页，直到抓取数量等于接口返回的 `total`。
5. 每页原始响应保存到 `data/raw/kaoyan_school_list/<batch_id>/page_<n>.json`。
6. 解析字段：`school_id`、`school_name`、`province_name`、`province_area`、`type_name`、`type_school_name`、`is_985`、`is_211`、`syl`、`is_zihuaxian`、`recruit_number`、`major_number`。
7. 输出标准化 CSV 到 `data/processed/universities/university_candidates_<batch_id>.csv`。
8. 对接口失败、超时、字段缺失进行日志记录。

#### 交付物

1. `src/crawlers/kaoyan_school_list_crawler.py`
2. 原始 JSON 文件
3. 标准化候选库 CSV
4. 爬虫运行日志

#### 测试方法

```powershell

Get-ChildItem -LiteralPath "D:\bigdatashixun\zhiyuan\data\processed\universities"
```

#### 完成标准

1. 能抓取 21 条重庆研招单位候选记录。
2. 能处理第一页 20 条和第二页 1 条数据。
3. 原始 JSON 和解析 CSV 都能保存。
4. 爬虫失败时有明确日志，不静默失败。
5. 不使用浏览器自动化作为常规爬虫方案。

| 图表展示 | 至少 2 个 ECharts 图表可正常展示 |
| 大数据链路 | 至少 1 个 Spark 分析任务可运行并输出结果 |
| AI 报告 | 大模型不可用时能返回模板报告 |
| 安全合规 | 页面和报告不出现录取承诺，不展示敏感个人信息 |
| 文档交付 | README、测试记录、开发步骤、详细设计和运行说明完整 |

## 9. 风险与处理建议

| 风险 | 影响 | 处理建议 |
| --- | --- | --- |
| 掌上考研接口变更 | 爬虫无法抓取候选库 | 保存最近一次原始 JSON，保留人工导入 CSV 降级方案 |
| 官方 PDF 格式复杂 | 提取准确率低 | 半自动提取后进入人工校对，不追求全自动 OCR |
| 样例数据不足 | 推荐结果不够丰富 | 第一版先保证 3 所学校、3 个专业门类可完整演示 |
| Hive 或 Spark 启动慢 | 影响演示 | 提前运行环境检查脚本，准备 Spark 输出截图或日志 |
| 大模型接口不可用 | 报告生成失败 | 模板报告作为默认可用方案 |
| 同学开发环境不一致 | 协作成本高 | 所有命令以 Docker 和 PowerShell 为准，README 写清楚 |
