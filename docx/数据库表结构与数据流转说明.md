## 一、数据库表分类（按数据流转顺序）

### 🔹 第一层：数据采集层（记录爬虫来源）

#### 1. `crawler_runs` - 爬虫运行批次
- 用途：记录每次爬虫运行的元数据
- 当前数据：2 条
- 数据来源：`source_registry.py` 自动写入

#### 2. `universities` - 招生单位候选库
- 用途：重庆研招单位基础信息（学校名称、层次、类型）
- 当前数据：21 条（重庆大学、西南大学等）
- 数据来源：掌上考研 V2 API → `source_registry.py` 入库
- 核验要求：`official_verified_status` 标记是否已官网核验

#### 3. `source_documents` - 来源资料索引
- 用途：记录每份原始资料（PDF、Excel、JSON）的来源和处理状态
- 当前数据：21 条（每所学校一条掌上考研候选来源）
- 数据来源：人工登记 + 爬虫自动登记
- 关键字段：`document_type`（资料类型）、`process_status`（处理状态）

---

### 🔹 第二层：基础数据层（学院、专业、学科评估）

#### 4. `departments` - 学院表
- 用途：存储学院名称（如"计算机学院"、"法学院"）
- 当前数据：0 条（待入库）
- 数据来源：待开发 - 从招生专业目录 PDF/Excel 提取
- 关联关系：`university_id` → `universities`

#### 5. `majors` - 专业表
- 用途：存储专业详情（专业代码、专业名称、研究方向、考试科目）
- 当前数据：0 条（待入库）
- 数据来源：待开发 - 从招生专业目录提取 + 掌上考研 V2 planDetail 数据
- 关联关系：`university_id` → `universities`、`department_id` → `departments`

#### 6. `subject_level_rates` - 学科评估等级表
- 用途：存储一级学科评估结果（A+、A、B+ 等）
- 当前数据：0 条（待入库）
- 数据来源：掌上考研 V2 schoolLevelRate API + 教育部官方学科评估
- 关联关系：`university_id` → `universities`

---

### 🔹 第三层：招生数据层（计划、分数线、录取）

#### 7. `enrollment_plans` - 招生计划表
- 用途：每年每个专业的招生计划、推免人数、统考名额
- 当前数据：0 条（待入库）
- 数据来源：招生专业目录 + 掌上考研 V2 planList 数据
- 关联关系：`year` + `university_id` + `major_id` 唯一

#### 8. `score_lines` - 复试分数线表
- 用途：国家线、院校线、专业线（总分线 + 单科线）
- 当前数据：0 条（待入库）
- 数据来源：掌上考研 V2 schoolScore API + 学校官网复试线公告
- 关键字段：`line_type`（国家线/院校线/专业线）

#### 9. `admission_records` - 拟录取记录表
- 用途：拟录取考生的初试成绩、复试成绩、总成绩（脱敏）
- 当前数据：0 条（待入库）
- 数据来源：学校官网拟录取名单 PDF/Excel（人工提取 + 校对）
- 隐私保护：`candidate_no_hash` 脱敏处理

---

### 🔹 第四层：分析结果层（ADS 应用数据服务）

#### 10. `major_statistics` - 专业统计结果表
- 用途：Spark 分析结果（最低分、平均分、最高分、热度评分）
- 当前数据：0 条（待入库）
- 数据来源：Spark 从 `admission_records`、`score_lines` 等表聚合计算
- 应用场景：推荐算法、图表展示

---

### 🔹 第五层：应用服务层（推荐、报告）

#### 11. `recommendation_logs` - 推荐日志表
- 用途：记录每次推荐请求和结果摘要
- 当前数据：0 条（待入库）
- 数据来源：Flask 推荐接口自动写入
- 关键字段：`trace_id`、`request_json`、`result_summary_json`

#### 12. `report_records` - 报告记录表
- 用途：保存生成的 AI 择校建议报告
- 当前数据：0 条（待入库）
- 数据来源：AI 报告生成接口
- 关键字段：`report_type`（llm/template）、`report_content`

---

### 🔹 第六层：监控与质量层

#### 13. `pipeline_runs` - 数据任务运行表
- 用途：记录数据清洗、同步、分析等 ETL 任务的执行情况
- 当前数据：0 条（待入库）
- 数据来源：各 ETL 脚本自动写入

#### 14. `data_quality_issues` - 数据质量问题表
- 用途：记录数据清洗过程中发现的异常（缺失、格式错误、重复）
- 当前数据：0 条（待入库）
- 数据来源：清洗脚本自动记录

## 二、数据流转全流程（从爬虫到推荐）

```
1. S03 爬虫采集
   └─> 掌上考研 V2 API
       └─> data/raw/kaoyan_v2/<batch_id>/
           ├─ school_list.json (21所学校)
           ├─ plan_list.json (招生计划)
           ├─ plan_detail.json (专业详情)
           ├─ score_line.json (分数线)
           └─ level_rate.json (学科评估)
       └─> data/processed/kaoyan_v2_integrated_<batch_id>.csv (方向级大合集)

2. S04 候选库入库
   └─> python -m src.data_pipeline.source_registry
       ├─ crawler_runs (写入 2 条)
       ├─ universities (写入 21 条)
       └─ source_documents (写入 21 条)

3. 官网资料人工收集（待完成）
   └─> 下载学校官网招生目录、复试线、拟录取名单 PDF/Excel
       └─> 保存到 data/raw/<school_name>/<year>/
       └─> 在 source_documents 表登记来源

4. 数据清洗与入库（待开发）
   └─> 提取脚本读取 PDF/Excel
       ├─> departments 表（学院）
       ├─> majors 表（专业）
       ├─> enrollment_plans 表（招生计划）
       ├─> score_lines 表（分数线）
       └─> admission_records 表（拟录取名单）

5. HDFS/Hive 同步（待开发）
   └─> CSV 导出 → HDFS → Hive 分区表
       └─> zhiyuan.universities
       └─> zhiyuan.majors
       └─> zhiyuan.score_lines
       └─> zhiyuan.admission_records

6. Spark 分析（待开发）
   └─> Hive 表 → Spark SQL/DataFrame 统计分析
       └─> major_statistics 表（最低分、平均分、热度）

7. Flask 推荐服务（待开发）
   └─> POST /api/recommend
       ├─ 读取 score_lines（判断过线）
       ├─ 读取 major_statistics（计算分差）
       ├─ 应用推荐规则（冲刺/稳妥/保底）
       └─> recommendation_logs 表（记录日志）

8. AI 报告生成（待开发）
   └─> POST /api/report/generate
       ├─ 读取 recommendation_logs
       ├─ 调用大模型 API（或模板降级）
       └─> report_records 表（保存报告）

9. 前端可视化（待开发）
   └─> GET /api/chart/line-trend
       └─> 读取 score_lines、major_statistics
       └─> ECharts 图表展示
```
