# 项目名称
-重庆高校志愿填报系统

# 项目背景
-我们是大学本科大数据专业大三的学生，小组共四人。我们现在要做一个实践项目，项目时长是三个星期，基于大数据平台对高考历年录取数据（院校投档线、专业录取线、一分一段表等公开数据）进行爬取、处理、存储、分析和可视化展示，并在此基础上实现志愿填报辅助功能：考生输入分数和科目类型后，系统给出"冲、稳、保"三档院校推荐，并由大模型生成个性化的志愿填报建议报告。

# 项目功能
1. 数据爬取：使用Python的爬虫库（如Scrapy或BeautifulSoup）从阳光高考、省教育考试院等公开网站上爬取历年录取数据，包括院校名称、招生年份、科目类型、批次、最低录取分、最低位次、招生计划数等，以及历年一分一段表和院校基本信息（所在省份、办学层次、院校类型等）。
2. 数据处理：对爬取到的数据进行清洗和预处理，包括去除重复数据、处理缺失值、院校名称标准化（统一全称与简称）、文理/物理历史科目组对齐，以及跨年份比较所需的"分数—位次"换算。
3. 数据存储：将处理后的数据存储到MySQL数据库中，并同步至Hadoop平台（HDFS/Hive），以便后续的分析和查询。
4. 数据分析：使用Spark和数据分析工具（如Pandas、NumPy）对存储的数据进行分析，挖掘录取数据中的趋势和规律，例如各院校分数线多年走势、位次波动情况（"大小年"现象）、同分数段可选院校数量变化、热门院校与专业分布等。
5. 可视化展示：使用数据可视化工具echarts将分析结果以图表的形式展示出来，包括院校分数线趋势图、省内院校分布图、专业热度排行等，帮助用户更直观地了解历年录取情况。
6. 志愿推荐：用户在网页端输入分数、省份和科目类型，系统先将分数换算为位次，再按照位次差划分"冲、稳、保"三个梯度，给出对应的院校推荐列表。
7. AI 建议报告：将推荐结果与考生分数位次信息交给大模型，自动生成一份个性化的志愿填报建议报告，内容包括志愿梯度是否合理、风险提示和专业选择建议；大模型只基于系统提供的真实数据生成内容。

# 项目技术栈
- 编程语言：Python = 3.12
- 爬虫库：Scrapy或BeautifulSoup
- 数据处理库：Pandas、NumPy
- 数据库：MySQL 8.0 redis 5.0.14
- 数据可视化工具：echarts
- Web框架：Flask
- 大数据平台：Hadoop3.3.6 hive 4.0.1 Spark3.5.1
