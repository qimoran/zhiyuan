## 二、S09 分数线评估模块更新

### 新增功能

#### 1. 国家线数据推导

**创新点**：
从掌上考研 V2 API 的 `score_diff_to_national` 字段反推国家线

**推导公式**：
```
国家线 = 专业线 - score_diff_to_national
```

**推导结果**：
- 242 条国家线数据（2024-2026 年）
- 覆盖 573 个专业类别
- 标准差为 0 的专业占比 >95%

**新增文件**：
- `scripts/derive_national_lines.py`（推导脚本）
- `sql/mysql/004_derived_national_lines.sql`（SQL 脚本）

#### 2. 数据粒度优势

| 类型 | 官方国家线 | 推导的国家线 |
|------|-----------|-------------|
| 粒度 | 13 个学科门类 | 242 个专业类别 |
| 示例 | 工学：265 分 | 计算机技术：273 分<br>电子信息：273 分<br>机械：273 分 |
| 优点 | 权威 | 更精确，直接匹配专业 |

## 三、详细设计文档更新

### 6.7 查询服务模块

**新增内容**：

#### 元数据服务接口设计

| 接口 | 说明 | 返回数据量 |
| --- | --- | --- |
| `list_major_categories()` | 专业门类列表 | 573 个 |
| `list_degree_types()` | 学位类型枚举 | 2 个 |
| `list_study_modes()` | 学习方式枚举 | 2 个 |
| `list_school_levels()` | 学校层次枚举 | 3 个 |

专业门类数据来源：
- `from_majors`：从 majors 表提取
- `from_score_lines`：从专业线提取
- `from_national_lines`：从国家线提取
- `combined`：合并去重（推荐使用）

### 6.8 分数线评估模块

**新增内容**：

#### 国家线数据来源说明

详细说明了：
1. 推导原理（从 `score_diff_to_national` 反推）
2. 推导结果（242 条，覆盖 573 个专业）
3. 数据质量（标准差 <0.01）
4. 数据特点（粒度更细，242 个专业类别 vs 13 个学科门类）
5. 推导脚本和 SQL 文件位置

#### 输出字段扩展

**新增字段**：
- `data_available`：是否有可用分数线
- `matched_line_count`：匹配到的分数线数量
- `reference_strategy`：参考策略（单一匹配或保守最高值）
- `candidate_lines`：候选分数线列表

#### 评估规则扩展

**新增规则**：
- 匹配到多条分数线时，按总分线最高值保守评估
- 数据完全缺失时返回 `data_available: false`
